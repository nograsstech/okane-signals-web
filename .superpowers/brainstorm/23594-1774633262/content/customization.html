What should users be able to customize?

What aspects of the favorites feature should be user-configurable?

A

Favorite card layout

Users choose what metrics appear on their favorite cards (e.g., win rate, return, Sharpe ratio, custom notes). Personalize what matters most.

Example
❤️ AAPL EMA ⚙️
Win: 52.1% | Sharpe: 1.8
"My best performer"
❤️ TSLA Bollinger ⚙️
Ret: +22.1% | Trades: 45
B

Favorite ordering and grouping

Users can sort favorites (by win rate, return, name) or group them (by ticker, strategy type, custom tags/folders).

Example
Sort:
📁 Tech Stocks
❤️ AAPL EMA
❤️ TSLA Bollinger
📁 Forex
C

Set favorite limits and alerts

Configure maximum number of favorites, get notified when favorites backtest results update, or set performance thresholds.

Settings
Notify on updates
D

Keep it simple for now

Just basic favorites functionality first. Can add customization features later based on user feedback. Ship faster, iterate sooner.

Pros

  • Faster to implement
  • Cleaner initial UX
  • Learn what users actually want

Cons

  • May need refactoring later
  • Less differentiated initially
