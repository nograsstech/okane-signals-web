Favorites limits and placement

A few practical details to nail down

Should there be a limit on the number of favorites?

A

No limit

Users can favorite as many strategies as they want. Simple and unrestricted.

B

Soft limit (e.g., 20)

Show a warning at 20, but allow more. Prevents abuse while keeping flexibility.

C

Hard limit (e.g., 10)

Force curation. Users must unfavorite to add more. Keeps favorites section focused.

Where should the favorites section appear?

1

Top of the strategy list page

Favorites appear above the main table on /strategy. First thing users see.

2

Collapsible section at top

Users can collapse the favorites section to save screen space. Remembers their preference.

3

Separate tab/section

"My Favorites" as a separate page or tab alongside the main strategy list. More separation.
