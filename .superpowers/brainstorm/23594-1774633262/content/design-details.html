Design: Favorite Strategies - Details

⚠️ Error Handling

Toggle Favorite Failures:
  • Optimistic UI update (heart fills immediately)
  • On error: rollback UI state, show toast notification
  • Retry mechanism: user can click again
Loading Favorites:
  • Show skeleton loading state for favorites cards
  • If API fails: show "Unable to load favorites" message with retry button
  • Graceful degradation: main table still works
Stale Favorites:
  • Backtest may be deleted but favorite remains
  • Show "No recent data" state on card with option to unfavorite
  • Don't auto-delete favorites (user's choice)

  • Client-side sorting/grouping - favorites fit in memory, no need for server-side

  • Group headers collapsible for large groups

🧪 Testing Strategy

Database Layer:
  • Test unique constraint prevents duplicate favorites
  • Test cascade delete when user is deleted
  • Test concurrent favorite/unfavorite operations
API Endpoints:
  • GET /api/user/favorites - returns only user's favorites
  • POST /api/user/favorites - creates favorite, validates params
  • DELETE /api/user/favorites - removes favorite, validates ownership
  • Test authentication required on all endpoints
UI Components:
  • FavoriteToggle: toggle updates optimistic state, calls API
  • FavoriteCards: renders loading state, empty state, populated state
  • Filter button: toggles between all/favorites view
  • Sort/group controls: update card arrangement
Integration:
  • Favorite configuration correctly matches latest backtest
  • Deleting a backtest doesn't delete the favorite
  • Notification toggle and favorite toggle work independently

🚀 Phase 1 Scope (MVP)

Included in initial release:
  • Database table and migration
  • API endpoints (GET, POST, DELETE)
  • Favorite toggle in table (heart icon column)
  • Favorites section at top of page (card grid)
  • "Favorites Only" filter button
  • Client-side sorting by win rate, return, name
  • Optimistic UI updates with error rollback
  • Persist sort preference in localStorage
Future enhancements (not in MVP):
  • Grouping by ticker/strategy/period
  • User notes on favorites
  • Favorites performance tracking over time
  • Notifications when favorite backtests update
  • Export/import favorites
  • Favorite folders/tags
